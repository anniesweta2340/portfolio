The Gemini Press Room an automated AI visibility desk

A live intelligence report that tracks where Gemini appears — and where it doesn't — across the AI-generated answers that shape how people discover products. Built in 48 hours at the Profound Hackathon.

The Question

When someone asks an AI assistant "what's the best tool for coding?" or "most private AI chatbot?" — Gemini either gets recommended or it doesn't. There's no dashboard for that. No rank tracker, no alert when a competitor displaces you, no way to calculate what being buried in AI answers actually costs. The Press Room asks: what would it look like to build one?

What I Built

  • A three-script Node.js pipeline that pulls live data from the Profound API, calculates revenue at risk per prompt, and uses the Claude API to draft counter-narratives
  • A competitive standings view showing Gemini vs. ChatGPT, Claude, Copilot, and others by share of voice and average position
  • A revenue-at-risk model that assigns a dollar value to each prompt where Gemini sits outside the top-3 recommendation band
  • A source intelligence layer showing which domains AI engines actually cite — so PR teams know exactly where to publish to move the answer
  • A Press Kit section where Claude drafts a counter-narrative for each underperforming prompt, grounded in the sources the engines already trust
  • A live refresh flow: paste a Profound API key and the report re-pulls all data client-side, no server needed

How the Pipeline Works

Three scripts run in sequence and write to a shared pressroom.json file that the front-end reads on load.

refresh.mjs
Calls Profound's visibility and citations endpoints to pull brand standings, worst-performing prompts, and top cited domains.
revenue_ladder.mjs
For each prompt where Gemini sits below position 3, calculates an inclusion gap and applies a conversion model to produce an annualized revenue-at-risk figure.
diagnose.mjs
Sends each underperforming prompt to the Anthropic Claude API. Claude returns a drafted counter-narrative grounded in the specific sources the engines already cite for that topic.
index.html
Single-file report. Reads static data by default. Live refresh overlays real-time Profound data when an API key is provided — the two modes coexist without a backend.

Revenue Model

The core formula: monthly mentions × inclusion gap × 15% consideration rate × 3% conversion × $240 annual seat value. The inclusion gap normalizes how far outside the top-3 band each prompt sits — a brand at position 13 loses more potential consideration than one at position 4. The output is an annualized revenue exposure figure, clearly labeled as an estimate based on assumed inputs, not disclosed data.

Every prompt where Gemini is buried below position 3 has a cost. The model makes that cost visible and sortable — so teams can prioritize by impact, not gut feel.

Key Decisions

01

Annualized, not monthly. The ACV is $240/seat/year. The mention window is 30 days. The output is how much a month of lost visibility costs projected over a year — labeled clearly to avoid the misleading impression of a recurring monthly loss.

02

Static fallback + live refresh. The report ships with pre-generated data and renders instantly without any API key. A live refresh mode overlays real-time data on top when a key is provided. The two modes coexist — no backend required.

03

Brand deduplication with weighted averages. The Profound data had case variants of the same brand (Mistral/MISTRAL, LLaMA/Llama, DeepSeek/Deepseek). Each pair was merged: mentions and share of voice summed, average position weighted by mentions — so standings reflect reality, not data artifacts.

04

Press Kit as the report's call to action. The section where Claude-generated counter-narratives live needed to read as a destination, not just another data table. A full-bleed dark panel separates it visually from the analysis above and signals: this is where you act.

What It Shows

The Press Room is a proof of concept for a new category of brand intelligence: automated AI-answer monitoring. As AI-generated responses displace traditional search results, the brands that win will be the ones who can measure their AI-search presence the way they currently manage SEO — and respond when they lose ground.

AI search isn't a future problem. Brands are already being recommended or ignored by AI answers at scale — and most have no infrastructure to know which, or why, or what to do about it.
